Round 2 Environment


The evaluations will be performed in Microsoft Azure Cloud. Each vendor will be provided an environment consisting of seven hosts on which to install their client software. The vendors will also have the option of installing server software onto a virtual machine (VM) already in the environment, or importing a VM if necessary. By default, the Azure VMs will be Standard B4MS, each with four vCPUs and 16GB memory. Each vendor will have full and complete administrative access to the hosts instantiated for them.

All machines will be in an isolated domain in their own resource group in their own virtual network. VPN access will enable connectivity to the environment, and passwords will be shared via out-of-band methods. There will be one VPN server per environment and vendors will RDP elsewhere within the environment. Hosts will be reachable only within the VPN. They will not have public IP addresses assigned to them via Azure, but they will be able to access the Internet. After VMs are created, an additional script will connect the VMs to the domain.

  • Test Range:
    • A Windows domain with one domain controller, one file server and five clients. All VMs will be the “Standard B4MS” instance, with four vCPUs and 16GB memory. The servers will run Windows Server with SKU: “2019-Datacenter” and the clients will run Windows 10 1903 with SKU “19h1-pro” or “1903-evd-o365pp.“
    • Windows Defender will be disabled for the evaluation.
  • Environment details are subject to change and additional information will be released upon Round 2 release.
